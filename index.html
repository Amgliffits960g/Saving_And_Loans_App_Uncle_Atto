Not connected yet, so data only lives on this device. Connect a free Supabase project once and every device signed into it — phone, tablet, desktop — will see the same clients and transactions live.

How to connect (2 minutes)
  1. Create a free project at supabase.com.
  2. In the SQL editor, run the snippet below once.
  3. Copy your Project URL and anon public key from Project Settings → API.
  4. Paste them into CONFIG.SUPABASE_URL and CONFIG.SUPABASE_ANON_KEY near the top of the script tag in this file, save, and re-open the app on every device.
create table clients (
  id text primary key,
  name text,
  phone text,
  photo text,
  loan_eligible boolean default false,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
create table transactions (
  id text primary key,
  client_id text references clients(id) on delete cascade,
  type text, amount numeric, note text,
  timestamp timestamptz, updated_at timestamptz default now()
);
alter table clients enable row level security;
alter table transactions enable row level security;
create policy "field agent access" on clients for all using (true) with check (true);
create policy "field agent access" on transactions for all using (true) with check (true);
alter publication supabase_realtime add table clients, transactions;

These policies allow anyone with your anon key to read/write — fine for a small trusted team, but add real authentication before wider rollout.
